5.스코프, 형변환

5-1 스코프1
```html
변수는 선언 위치에 따라 지역 변수, 멤버 변수와 같이 분류된다.
지역 변수는 특정 지역에서만 사용할 수 있는 변수이다. 지역은 코드블록 {}이다.
생존 범위만 맞으면 다 접근할 수 있다.
지역 변수는 본인의 코드 블록 안에서만 생존한다.
```
5-2 스코프2
```html
스코프 존재 이유: 
1. 임시 변수(혹은 불필요한 변수)가 넓은 범위에 선언되어 있으면? 넓은 범위의 코드 블록이 종료 될 때까지 메모리에 유지된다.
따라서 불필요한 메모리가 낭비된다. 변수의 스코프를 줄일 수록 효율적인 메모리 사용이 가능하다.
2. 코드 복잡성이 증가할 수 있다. 좋은 코드는 단순한 코드이고 다른 사람이 봤을 때 이해하기 쉬워야 한다. 스코프가 불필요하게 넓으면 다른 사람이 볼때 변수를 더 생각해야한다.
정리: 변수는 꼭 필요한 범위로 한정하자. 메모리를 효율적으로 사용할 수 있고, 더 유지보수하기 좋은 코드를 만들 수 있다. 좋은 프로그램은 무한한 자유가 있는 프로그램이 아니라 적절한 제약이 있는 프로그램이다.
```
5-3 자동 형변환
```html
작은 범위에서 큰 범위로는 당연히 큰 값을 넣을 수 있다. int > long > double
작은 범위에서 큰 범위로의 형변환은 자동으로 일어나기 때문에 자동 형변환, 묵시적 형변환이라고 한다.
```
5-4 명시적 형변환
```html
큰 범위에서 작은 범위 대입은 명시적 형변환이 필요하다.
double을 int로 넣을 경우 소숫점 자리가 손실될 수 있다. 큰 걸 작은 것에 넣으려고 하니까 발생하는 것이다. 이는 큰 오류를 범할 수 있다.
intValue = (int) doubleValue; : (int)를 적음으로써 개발자가 명시적으로 형변환 하려고 한다.
int의 범위에 벗어나는 값을 형변환하면 오버플로우 에러가 발생한다. 에러 결과를 계산하지 말고 아예 안생기게 처음부터 조심하자
```
5-5 계산과 형변환
```html
같은 타입끼리의 계산은 같은 타입의 결과를 낸다.
서로 다른 타입의 계산은 큰 범위로 자동 형변환이 일어난다.
double div3 = (double) 3 / 2; : 3이 double로 캐스팅 된다. 2는 int이므로 더 큰 double로 자동 형변환 된다. double/double이므로 결과값은 double이 된다.
```
