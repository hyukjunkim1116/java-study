4. IO 활용

4-1. ObjectStream
```html
ObjectStream을 사용하면 메모리에 보관되어 있는 회원 인스턴스를 파일에 편리하게 저장할 수 있다.
객체 직렬화는 메모리에 있는 객체 인스턴스를 바이트 스트림으로 변환하여 파일에 저장하거나 네트워크를 통해 전송할 수 있도록 하는 기능이다. 나중에 역직렬화를 통해 원래의 객체로 복원할 수 있다.
직렬화하려는 클래스는 반드시 Serializable 인터페이스를 구현해야 한다.
객체 직렬화는 1990년대에 등장한 기술로 초창기에는 인기가 있었지만 여러 단점이 드러났다. 또한 대안 기술이 등장하면서 사용하지 않게 되었다.
```

4-2 XML, JSON, 데이터베이스

```html
객체 직렬화는 버전 관리가 어렵고 플랫폼에 종속적이다. 또한 성능이 안좋고 유연성이 부족하다.
이 문제를 해결하기 위해 XML이라는 기술이 인기를 끌었다. XML은 텍스트 기반의 데이터 교환 형식으로 데이터를 저장하고 전송하는 데 사용된다.
하지만 XML은 데이터를 저장하고 전송하는 데 사용하기에는 너무 복잡하다. 그래서 JSON이라는 기술이 등장했다. JSON은 XML보다 간단하고 가볍다.
JSON은 표준 데이터 교환 포맷으로 자리잡았다.
**정리**
자바 객체 직렬화는 대부분 사용하지 않는다.
JSON이 사실상 표준이다. JSON을 먼저 고려하자.
성능 최적화가 매우 중요하다면 Protobuf, Avro 같은 기술을 고려하자. (대부분 JSON만 사용해도 충분하다)

회원과 같은 구조화된 데이터를 보관하는 방법에 대해 고민해보자.
파일에 데이터를 직접 저장하는 방식은 몇가지 한계점이 있다.
1. 데이터의 무결성을 보장하기 어렵다. 여러 사용자가 동시에 파일을 수정하거나 접근하려고 할때 데이터의 충돌이나 손상 가능성이 높아진다. 이런 경우 데이터의 일관성을 유지하는 것이 매우 어렵다.
2. 데이터 검색과 관리가 비효율적이다. 파일에 저장된 데이터는 특정 형식 없이 단순히 저장될 수 있기 때문에 필요한 데이터를 빠르게 찾는데 많은 시간이 소요될 수 있다. 특히 데이터의 양이 방대해질수록 검색 속도는 급격히 저하된다.
3. 보안 문제가 있다. 파일 기반 시스템에서는 민감한 데이터를 안전하게 보호하기 위한 접근 제어와 암호화 등이 충분히 구현되지 않을 수 있다. 결과적으로 데이터 유출이나 무단 접근의 위험이 커질 수 있다.
4. 대규모 데이터의 효율적인 백업과 복구가 필요하다.
이런 문제점들을 하나하나 해결해가면서 발전한 서버 프로그램이 데이터베이스이다.
실무에서는 대부분 대부분의 데이터를 파일에 저장하지 않고 데이터베이스에 저장한다.
```
