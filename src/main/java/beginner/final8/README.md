8.final

8-1: final 변수와 상수
```html
final은 말 그래도 끝이라는 뜻이다. 더이상 값 변경 불가.
class, method를 포함한 여러 곳에 붙을 수 있다.
지역변수에 쓰면 한번만 설정가능
클래스의 인스턴스 변수에 쓰면 생성자 호출시 한번만 설정 가능
필드에서 초기화 하면 이미 설정 되었기 때문에 생성자를 통해서도 초기화 불가능
static 에도 final 선언 가능
인스턴스 변수에 파이널 쓰면 힙 영역에 자동으로 같은 값이 생성된다. 이는 메모리 낭비를 유발하고 중복이다.
따라서 이런 경우 static을 쓴다.
static final ... 을 쓰면 jvm상에서 하나만 존재한다. 이는 중복과 메모리 비효율 문제를 모두 해결한다.
```

8-2: 상수
```html
상수는 단 하나만 존재하는 변하지 않는 고정된 값이다. static final을 쓴다.
이런 경우 변수 이름은 영어대문자+ 언더스코어를 이용해서 작명하고 필드에 직접 접근해서 사용한다.
```

8-3: final 변수와 참조
```html
final 변수를 사용하면. 기본형 변수에 사용하면 값을 변경할 수 없고 참조형 변수에 사용하면 참조값을 변경할 수 없다.
참조형 변수에 사용하면 참조값을 다른 값으로 변경할 수 없다. 참조값 클래스 내부 필드는 변경 가능
```
