4.생성자

4-1 생성자가 필요한 이유
```html
객체 생성 시점에 어떤 작업을 하고 싶으면 생성자 이용!
초기값 설정할떄!
```

4-2 this
```html
멤버 변수와 메서드의 매개변수 이름이 같으면 매개변수가 우선순위를 가진다.
멤버 변수에 접근하려면 앞에 this.를 붙여주면 된다.
this.name= name : 1. 오른쪽 name은 매개변수 접근. 2. this.은 인스턴스 자신의 참조값. 인스턴스의 멤버 변수 접근
name = name은 둘다 매개변수를 뜻하므로 멤버변수의 값이 변경되지 않는다.
this 생략 가능. 이때 가까운 지역변수를 찾고 없으면 다음으로 멤버 변수를 찾는다. 없으면 오류 발생.
```

4-3 생성자-도입
```html
객체 생성하고 초기값 할당해야 하는 경우가 많다. 이런 경우 더 편리하기 위해 생성자를 사용한다.
생성자는 이름은 클래스 이름과 같아야 한다. 생성자는 반환 타입이 없다. 나머지는 메서드와 같다.
생성자는 인스턴스를 생성하고 나서 즉시 호출된다.
new 클래스이름(생성자에 맞는 인수 목록)
```

4-4 생성자 장점
```html
생성자가 없으면 작업 수행을 위해 메서드 호출을 한번 더 해야했다.
생성자 덕분에 객체 생성과 생성 직후 필요한 작업을 한번에 처리 가능
생성자의 진짜 장점은 생성자을 만들었다면 직접 정의한 생성자를 반드시 호출해야 한다.
생성자를 메서드 오버로딩처럼 여러개 정의 가능한데, 하나만 호출하면 된다.
생성자를 호출하지 않으면 컴파일 오류가 뜨는데 덕분에 초기화 과정을 안까먹을 수 있다.
따라서 생성자를 사용하면 필수값 입력을 보장할 수 있다.
"좋은 프로그램은 무한한 자유도가 주어지는 프로그램이 아니라 적절한 제약이 있는 프로그램이다"
```

4-5 기본 생성자
```html
MemberInit() {} : 기본 생성자. 안만들어도 컴파일러가 자동으로 만들어줌. 따로 만든 생성자가
하나라도 있으면 자바는 기본 생성자를 만들지 않는다.
```

4-6 생성자 오버로딩과 this
```html
생성자 오버로딩할때 겹치는 부분을 this()로 해결할 수 있다.
this()는 생성자 내부에서 다른 생성자를 호출할 수 있다.
this()는 생성자 코드의 첫줄에만 작성할 수 있다.
```
