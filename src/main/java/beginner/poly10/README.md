10-1: 다형성
```html
객체지향 프로그래밍의 대표적인 특징은 캡슐화,상속,다형성이다. 그 중에서 다형성은 객체지향 프로그래밍의 꽃이다.
다형성은 한 객체가 여러 타입의 객체로 취급될 수 있는 능력이다.
```
10-2: 다형적 참조
```html
부모 변수가 자식 인스턴스 참조 : 다형적 참조
부모 타입은 자식 타입을 담을 수 있지만 반대는 불가능
자바에서 부모 타입은 자신은 물론이고 모든 자식 타입을 참조할 수 있다. 이를 다형적 참조라고 한다.
다형적 참조시 호출자의 타입은 부모타입이므로 자식타입의 메서드를 호출 하면 컴파일 오류가 생긴다 이때 캐스팅을 활용하면 참조 가능하다.
```

10-3: 다형성과 캐스팅
```html
업캐스팅: 부모타입으로 변경
다운캐스팅: 자식타입으로 변경
((Child) poly).childMethod(); : 일시적 다운캐스팅
업캐스팅은 생략 가능. 자바에서 자동으로 넣는다. 
다운 캐스팅은 심각한 런타임 오류가 발생할 수 있기 때문에 명시해야하고 반대로 업 캐스팅은 필요없다.
업 캐스팅시 메모리상에 부모와 자식 인스턴스가 모두 존재해서 안전함. 반대로 다운 캐스팅 할때는 자식 인스턴스가 메모리에 없을 수 있다.
```

10-4 instanceOf
```html
a instanceof b : b의 타입에 a의 타입이 들어갈 수 있는지 확인
자식 instanceof 부모 : true. 이때 참조값으로 확인
Parent parent = new Child();
parent instanceof Child : true
if(parent instanceof Child child) : true이면 child 변수 생성
```

10-5 다형성과 메서드 오버라이딩
```html
오버라이딩 된 메서드가 항상 우선권을 가진다.
변수는 오버라이딩 되지 않고 메서드는 오버라이딩 된다.
변수의 타입으로 먼저 찾아서 확인한다. 만약 오버라이딩 되어 있으면 된걸 실행한다.(자식으로감)
Parent parent = new Child();
System.out.println("p -> c");
System.out.println("value = parent.value"); : 변수 오버라이딩 x, 변수 타입이 부모이므로 부모의 밸류를 찾음
parent.method(); : 메서드는 오버라이딩 가능. 변수 타입이 부모이므로 부모로 가서 찾지만 자식이 오버라이딩했으면 자식껄로 실행

```
