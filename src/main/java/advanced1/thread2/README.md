2. 스레드 생성과 실행

2-1 스레드 시작
```html
자바 메모리 구조
메서드 영역 : 프로그램을 실행하는데 필요한 공통 데이터. 프로그램의 모든 영역에서 공유.
클래스 정보, 스태틱, 런타임 상수 풀 저장
스택 영역 : 자바 실행시 메서드 호출할 때 마다 하나의 스택 프레임이 쌓이고 메서드가 종료되면 해당 스택 프레임이 제거된다.
힙 영역 : 객체와 배열이 생성되는 영역이다. 가비지 컬렉션이 이루어지는 주요 영역이다.
Thread 클래스를 상속 받거나 Runnable 인터페이스를 구현하면 스레드가 생성된다.
스레드는 순서와 실행 기간을 모두 보장하지 않는다. 이것이 바로 멀티스레드다!
```

2-2 데몬 스레드
```html
사용자 스레드 : 프로그램의 주요 작업을 수행한다.
데몬 스레드 : 백그라운드에서 보조적인 작업을 수행한다.
```

2-3 Thread 상속 vs Runnable 구현
```html
스레드를 사용할 떄에는 Thread를 상속 받는 방법보다 Runnable 인터페이스를 구현하는 방식을 사용하자
Thread 클래스 상속 방식은 run() 메서드만 재정의하면 되므로 간단하게 구현 가능
하지만 자바는 단일 상속만을 허용하므로 이미 다른 클래스를 상속받고 있는 경우 Thread 클래스를 상속 받을 수 없다.
그리고 인터페이스를 사용하는 방법에 비해 유연성이 떨어진다
Runnable 인터페이스를 구현하면
상속으로부터 자유롭고 코드가 분리될 수 있다.
그리고 여러 스레드가 동일한 Runnable 객체를 공유할 수 있어 자원 관리를 효율적으로 할 수 있다.
반면 코드가 복잡해 질 수 있다는 단점이 있다.
```
