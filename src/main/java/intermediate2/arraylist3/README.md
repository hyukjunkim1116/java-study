3. ArrayList

3-1 배열과 인덱스
```html
배열의 경우 인덱스를 사용하면 한번의 계산으로 매우 효율적으로 자료의 위치를 찾을 수 있다. 배열에서 인덱스를 사용하는 경우
데이터가 아무리 많아도 한 번의 연산으로 필요한 위치를 찾을 수 있다.
배열의 검색은 배열의 크기가 n이면 연산도 n만큼 필요하다.
int[]
```

3-2 빅오(O) 표기법
```html
빅오 표기법은 알고리즘의 성능을 분석할 때 사용하는 수학적 표현 방식이다. 데이터 양의 증가에 따른 성능의 변화 추세를
이해하는 것이다.
배열의 인덱스 사용 : O(1)
배열의 순차 검색 : O(n)
```

3-3 배열에 데이터 추가
```html
데이터를 중간에 추가하면 기존 데이터가 오른쪽으로 한 칸씩 이동해야 한다. O(n)
배열의 한계 : 배열은 기본적인 자료구조이고 인덱스를 사용하면 최고의 효율이 나온다. 하지만 배열의 크기를 배열 생성 시점에 미리 정해야 한다.
따라서 동적으로 언제든지 길이를 늘리고 줄일 수 있는 자료 구조가 있으면 편리할 것이다.
```

3-4 배열 리스트
```html
배열은 길이를 동적으로 변경할 수 없고 데이터를 추가하기 불편하다. 직접 데이터를 오른쪽으로 한 칸씩 밀어야하는 코드를 짜야한다.
List : 리스트, 순서가 있고 중복을 허용하는 자료 구조.
배열은 순서가 있고 중복을 허용하지만 크기가 정적이다
리스트는 순서가 있고 중복을 허용하지만 크기가 동적으로 변할 수 있다.
ArrayList는 리스트 자료 구조를 사용하면서 내부 데이터는 배열에 보관하는 것이다.
데이터 추가 :
    마지막에 추가 : O(1)
    앞, 중간에 추가 : O(n)
데이터 삭제 :
    마지막에 삭제 : O(1)
    앞, 중간에 삭제 : O(n)
인덱스 조회 : O(1)
데이터 검색 : O(n)
배열 리스트는 마지막에 데이터를 추가하거나 마지막에 있는 데이터를 삭제할 때는 매우 빠르고 중간에 데이터를 추가하거나 삭제할 때는 O(n)으로 느리다
데이터를 순서대로 입력하고 순서대로 출력하는 경우에 가장 효율적이다.
LinkedList는 반면에, 중간에 데이터를 추가하거나 삭제할 때는 효율적이다.
```
