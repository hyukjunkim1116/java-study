4. LinkedList

4-1 배열리스트의 단점
```html
배열은 필요한 배열의 크기를 미리 확보해야 한다. 
따라서 나머지 공간은 사용되지 않고 낭비된다.
중간에 데이터를 추가하거나 삭제하는 경우 많은 데이터를 이동해야 하기 때문에 성능이 좋지 않다.
따라서 노드를 만들고 서로 연결해서 순서를 보장하면 메모리 낭비가 없고 중간에 추가하거나 삭제할때 효율적이다.
이것이 링크드 리스트의 원리이다.
```

4-2 노드
```html
노드는 내부에 데이터와 다음 노드에 대한 참조를 가지고 있다.
각각의 노드가 참조에 의해 연결되어 있다.
데이터를 추가할 때 동적으로 필요한 만큼의 노드만 만들어서 연결하면 된다.
따라서 배열과 다르게 메모리를 낭비하지 않는다.
이런 자료 구조를 연결리스트(LinkedList)라고 한다.
배열리스트든 연결리스트든 둘다 순서가 있고 중복을 허용하는 자료 구조이다.
```

4-3 연결 리스트와 빅오
```html
**직접 만든 배열 리스트와 연결 리스트의 성능 비교 표**
기능

        배열 리스트 연결 리스트
인덱스 조회     O(1) O(n)
검색          O(n) O(n)
앞에 추가(삭제) O(n) O(1)
뒤에 추가(삭제) O(1) O(n)
평균 추가(삭제) O(n) O(n)
데이터를 조회할 일이 많고 뒷 부분에 데이터를 추가한다면 배열 리스트가 더 좋은 성능을 제공한다.
반면, 앞쪽 데이터를 추가하거나 삭제할 일이 많다면 연결 리스트를 사용하는 것이 더 좋은 성능을 제공한다.
```
