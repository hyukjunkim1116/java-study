7. HashSet

7-1. 해시코드
```html
해시 인덱스를 사용하려면 데이터의 값을 배열의 인덱스로 사용해야 한다.
배열의 인덱스는 숫자만 사용 가능한데, 문자열 데이터를 저장할때는 해시 인덱스를 어떻게 할까?
모든 문자는 본인만의 고유한 숫자로 표현 가능하다. 아스키 코드표를 사용한다.
해시 함수는 임의의 길이의 데이터를 입력으로 받아 고정된 길이의 해시값을 출력하는 함수이다.
같은 데이터를 입력하면 항상 같은 해시 코드가 출력된다.
다른 값을 입력해도 같은 해시 코드가 출력 될 수 있고 이것을 해시 충돌이라고 한다.
해시 코드는 데이터를 대표하는 값이다.
해시 인덱스는 데이터의 저장 위치를 결정하는데, 해시 코드를 사용해서 만든다.
해시코드:데이터를 대표하는 값
해시함수:해시코드를 생성하는 함수
해시인덱스:해시코드를 사용한 데이터의 저장 위치를 결정하는 값
따라서, 문자 데이터를 사용할 경우에도 해시 함수를 사용해서 정수 기반의 해시코드로 변환한 덕분에,
해시 인덱스를 사용할 수 있게 되었다.
해시 코드만 정의할 수 있다면 어떤 객체든지 해시 인덱스를 사용할 수 있다.
```

7-2 자바의 hashCode()
```html
자바는 모든 객체가 자신만의 해시 코드를 표현할 수 있는 기능을 제공한다.
Object.hashCode() 메서드이다.
객체의 인스턴스가 다르면 해시 코드도 다르다. 객체의 참조값을 기반으로 해시 코드를 생성하기 때문이다.
자바의 기본 클래스들을 대부분 내부 값을 기반으로 코드를 구할 수 있도록 hashCode() 메서드를 재정의 했다.
따라서 데이터의 값이 같으면 같은 해시코드를 반환한다.
정수를 반환하기 때문에 마이너스 값이 나올 수 있다.
참고 : == 동일성, 같은 객체인지 참조값 확인 // equals() 동등성, 논리적으로 동등한지 확인
따라서 내가 직접 만든 Member객체는 id값이 같아도 다른 해시코드를 반환한다.
이를 hashCode()메서드를 오버라이딩해서 id기준으로 같은 해시코드를 반환하게 수정하면 된다.
정리 :
    자바가 기본으로 제공하는 클래스 대부분은 hashCode()를 재정의함
    객체를 직접 만들어야 하면 재정의하면 된다.
    그러면 모든 종류의 객체를 해시 자료구조에 보관 가능
    해시 자료구조에 데이터를 저장하는 경우 hashCode()를 구현해야 한다.
```

7-3 equals()
```html
equals는 해시 충돌이 날때 내부 배열을 하나하나 비교할때 사용한다.
따라서 해시 자료구조를 사용할 때에는 hashCode()는 물론이고 equals()도 반드시 재정의 해야 한다.
자바가 제공하는 기본 클래스는 이미 재정의 해두었다.
해시코드를 재정의 하지 않으면 다른 해시 인덱스에 데이터가 저장되고
equals를 재정의 하지 않으면 중복체크에 실패하고 데이터 검색이 안된다.
따라서 해시 자료구조를 사용할 때에는 hashCode()는 물론이고 equals()도 반드시 재정의 해야 한다.
```
