2. 제네릭2

2-1 타입 매개변수 제한 이유
```html
상속과 다형성을 활용하면 코드 재사용과 타입 안정성 두마리 토끼를 둘다 잡을 수 없다.
그래서 제네릭을 활용해서 상속을 구현하면 Object가 제공하는 메서드만
호출할 수 있다. 그래서 내가 만든 클래스의 메서드를 사용할 수 없다.
그리고 Object가 부모이기 때문에 전혀 관계 없는 타입을 인자로 전달할 수 있는 문제가 있다.
```

2-2 타입 매개변수 제한
```html
<T extends Animal> 로 타입 매개변수를 상한하면 인자에 Animal과 그 자식만 받을 수 있게 된다. 따라서 Animal이 제공하는
메서드를 사용 가능하게 된다.
```

2-3 제네릭 메서드
```html
제네릭 타입과 제네릭 메서드는 둘다 제네릭을 사용하기는 하지만 서로 다른 기능을 제공한다.
제네릭 메서드는 특정 메서드 단위로 제네릭을 도입할 때 사용한다.
<T> T genericMethod(T t)
다이아몬드를 사용해서 <T>와 같이 타입 매개변수를 적어준다. 실제 호출하는 시점에 타입을 정하고 호출한다.
제네릭 메서드는 인스턴스 메서드와 static 메서드에 모두 적용할 수 있다.
제네릭 타입은 static 메서드에 타입 매개변수를 사용할 수 없다. 클래스 단위로 작동하기 때문에 제네릭 타입과는 무관하다.
인스턴스 메서드에는 제네릭 타입 적용 가능
```

2-4 타입 매개변수 제한
```html
제네릭 메서드도 제네릭 타입과 마찬가지로 타입 매개변수를 제한할 수 있다. 제네릭 타입과 같음
제네릭 메서드의 타입 추론 : GenericMethod.<Integer>genericMethod(1) >> GenericMethod.genericMethod(1)
타입을 컴파일러가 대신 처리하기 때문에 타입을 전달하지 않는 것 처럼 보일 수 있지만 실제로는 타입 인자가 전달 된다.
```

2-5 제네릭 우선순위
```html
인스턴스 메서드에서 제네릭 타입과 제네릭 메서드의 타입 매개변수가 같으면 우선순위는
제네릭 메서드 이다. 그래서 직접 명시하지 않으면 Object로 취급된다. 그래서 모호하지 않게 둘중 하나를 다른 이름으로 바꾸자
```

2-6 와일드카드1
```html
와일드카드는 제네릭 타입이나 메서드를 선언하는 것이 아니라 이미 만들어진 제네릭 타입을 활용할때 사용한다.
와일드카드 ?는 모든 타입을 다 받을 수 있다.
? == <? extends Object>라고 볼 수 있다.
제네릭 메서드의 타입 매개변수에 특정 시점에 타입 인자를 전달해야 한다. 이는 복잡한 과정이다.
하지만 와일드카드를 쓰면 단순히 매개변수에 제네릭 타입을 넣으면 된다. 단순히 일반 메서드에 제네릭 타입을 받을 수 있는 매개변수가
하나 있는 것 뿐이다. 따라서 제네릭 타입이나 메서드를 정의해야하는 상황이 아니라면 와일드카드를 사용하자
와일드카드에도 상한 제한을 걸 수 있다.
```

2-7 타입 매개변수가 꼭 필요한 경우
```html
제네릭 타입이나 메서드를 사용하면 전달한 타입을 명확하게 반환할 수 있다.
반면 와일드카드로 정의하면 부모의 타입으로 반환된다.
와일드카드는 이미 만들어진 제네릭 타입을 전달 받아서 활용할 때 사용한다. 따라서 메서드의 타입들을
타입 인자를 통해 변경할 수 없다. 일반적인 메서드에 사용한다고 생각하자.
제네릭 타입이나 메서드가 필요하면 <T>를 사용하고 아니면 와일드카드를 사용하자
```

2-8 하한 와일드카드
```html
<? super Animal> : Animal 타입을 포함한 상위 타입만 입력 받을 수 있다.
```

2-9 타입 이레이저
```html
제네릭은 컴파일 단게에서만 사용되고 이후에는 제네릭 정보가 삭제된다.
타입 매개변수를 제한하면 제한한 타입으로 코드를 변경한다.
컴파일시 자바가 타입인자로 캐스팅하는 코드를 넣어준다.
제네릭은 단순히 개발자가 직접 캐스팅하는 코드를 컴파일러가 대신 처리해 주는 것이다.
컴파일 시점에만 존재하고 런타임 시에는 제네릭 정보가 지워지는데 이것을 타입 이레이저 라고 한다.
```

2-10 타입 이레이저의 한계
```html
instanceof 또는 new를 사용하면 T가 Object로 컴파일 된다. 이는 개발자가 의도한 것과 다르다.
따라서 자바는 타입 매개변수에 instanceof와 new를 허용하지 않는다.
```

2-11 정리
```html
실무에서 제네릭을 사용해서 무언가를 만드는 일은 드물다. 그래서 이미 만들어진 코드의 제네릭을 읽고 이해하는 정도면 충분하다.
공변, 반공변 같은 더 어려운 개넘들도 있다. 하지만 복잡한 라이브러리나 프레임워크를 설계하지 않을 거라면 이런 개념들을
꼭 이해한 필요는 없다. 필요하다고 느끼면 따로 공부하자.
제네릭은 컬렉션 프레임워크에서 가장 많이 사용된다. 따라서 컬렉션 프레임워크를 통해서 제네릭이 어떻게 활용되는지 자연스럽게 학습하자
```
