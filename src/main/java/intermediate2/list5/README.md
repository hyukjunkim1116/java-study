5. List

5-1 List
```html
순서가 있고 중복을 허용하는 자료 구조를 리스트라고 한다.
```

5-2 시간 복잡도와 실제 성능
```html
이론적으로 연결리스트의 평균 추가 연산은 배열리스트보다 더 빠를 수 있다.
하지만 실제 성능은 요소의 순차적 접근 속도, 메모리 할당 및 해제 비용, CPU 캐시 활용도 등
더 다양한 요소에 의해 영향을 받는다.
배열리스트는 요소들을 메모리 상에서 연속적으로 위치하여 CPU 캐시 효율이 좋고
메모리 접근 속도가 빠르다. 반면 연결리스트는 각 요소가
별도의 객체로 존재하고 다음 요소의 참조를 저장하기 때문에 CPU 캐시 효율이 떨어지고
메모리 접근 속도가 상대적으로 느릴 수 있다.
대부분의 경우 배열 리스트가 성능상 유리하기 때문에
실무에서는 주로 배열 리스트를 기본으로 사용한다. 만약에 데이터를 앞쪽에서 자주 추가하거나 삭제할 일이 있다면 연결 리스트를 고려하자.
```

5-3 자바 리스트
```html
<interface>Collection 인터페이스를 List 인터페이스가 구현하고 이를 배열리스트와 연결리스트가 상속받는 형태이다.
컬렉션 인터페이스는 자바에서 다양한 데이터그룹(컬렉션)을 다루기 위한 메서드를 정의한다.
List 인터페이스는 컬렉션 프레임워크의 일부이다. 객체들의 순서가 있는 컬렉션을 나타내며 같은 객체의 중복을 허용한다.
배열과 비슷하지만 크기가 동적으로 변하는 컬렉션을 다룰 때 유연하게 사용 가능하다.
```

5-4 자바의 배열리스트
```html
배열을 사용해서 데이터를 관리한다.
기본 크기는 10이다.
크기를 넘어가면 배열을 50% 증가시킨다.
메모리 고속 복사 연산을 사용한다.
```

5-5 자바의 연결리스트
```html
이중 연결 리스트 구조이다.
첫 번째 노드와 마지막 노드 둘다 참조한다. 역방향으로 조회 가능하다.
```

5-6 자바의 리스트와 빅오
````html
기능 배열 리스트 연결 리스트
앞에 추가(삭제) O(n) - 106ms O(1) - 2ms
평균 추가(삭제) O(n) - 49ms O(n) - 1116ms
뒤에 추가(삭제) O(1) - 1ms O(1) - 2ms
인덱스 조회 O(1) - 1ms O(n) - 평균 439ms
검색 O(n) - 평균 104ms O(n) - 평균 473ms
자바의 배열 리스트는 메모리 고속 복사를 활용하기 때문에 성능이 최적화된다.
대부분의 경우 배열 리스트가 성능상 유리하다. 실무에서는 기본적으로 배열 리스트를 사용한다.
하지만 데이터를 앞쪽에서 자주 추가하거나 삭제할 일이 있다면 연결 리스트를 고려하자.
int[]는 배열. List는 리스트 ArrayList는 배열리스트, LinkedList는 연결리스트
참고로 리스트는 배열과 달리 자료 구조의 크기가 동적으로 증가한다. 따라서 배열처럼 입력 가능한 크기를 미리 정하지 않아도 된다.
리스트는 size() 메서드를 통해 입력된 데이터의 크기를 제공한다.
````
