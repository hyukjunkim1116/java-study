1. 제네릭

1-1 제네릭이 필요한 이유
```html
제네릭을 사용하지 않고 다형성을 활용하면 실수로 원하지 않는 타입이 들어갈 수 있는
문제가 발생한다. 그리고 반환 시점에서도 위험한 다운 캐스팅을 해야 한다. 제네릭은
이 문제들을 모두 해결할 수 있다.
```

1-2 제네릭 클래스
```html
<>를 사용한 클래스를 제네릭 클래스라고 한다. 클래스명 오른쪽에 선언한다. <T>에서 T를 타입 매개변수라고 한다.
생성 시점에 지정한 타입으로 변한다.
자바 컴파일러가 우리가 입력한 타입 정보를 기반으로 이런 코드가 있다고 가정하고 컴파일 과정에서 타입 정보를 반영한다.
코드 주변에 추론할 수 있게 타입 정보가 있으면 타입 추론이 된다. 타입 추론은 자바가 스스로 타입 정보를 추론해서
개발자가 타입 정보를 생략할 수 있는 것이다.
```

1-3 제네릭 용어와 관례
```html
제네릭의 핵심은 사용할 타입을 미리 결정하지 않는다는 점이다.
클래스를 정의할때 결정하는 것이 아니라 실제 사용하는 생성 시점에 타입을 결정한다.
제네릭은 메서드의 매개변수(파라미터)와 인자의 관계와 비슷하게 작동한다.
차이점은 메서드의 매개변수는 사용할 값을 나중으로 미루는 것이고 제네릭의 타입 변수는 사용할 타입을 나중으로 미루는 것이다.
GenericBox<T>에서 GenericBox<T> 는 제네릭 타입, T는 타입 매개변수, GenericBox<Integer>에서 Integer를 타입 인자 라고 한다.
관례 :
    E - Element
    K - Key
    N - Number
    T - Type
    V - Value
    S,U,V - etc...
한번에 여러 타입 매개변수 선언 가능. 타입 인자로 기본형은 사용 불가하고 래퍼 클래스 사용 가능
로우 타입 : <>를 지정하지 않는 것은 내부 타입 매개변수가 Object로 사용된다고 알면 된다. 이는 예전 자바 버전의 코드와 호환하기 위해 지원하는 중이다.
따라서, Object 타입을 사용해야 한다면 타입 인자를 Object로 명시하자.
```
